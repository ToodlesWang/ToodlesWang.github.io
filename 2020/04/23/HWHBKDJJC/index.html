为什么我要搭建博客

互联网时代带来的信息碎片化使信息的价值呈两极化发展：简短精炼的语句需要被投入的阅读时间及用来走马观花消耗的精力与其核心优势成正比；千篇万字的论文式文章愈发被大众抛弃，使其传播范围不如前者之广泛，受众群体易单一化。而作为注定与互联网绑定一生的网民但又同时无时无刻不在接收信息的我需要辨别好坏信息的能力，信息以毫秒为单位在我面前不断刷新，还未来得及消化上一条，下一条便接踵而至，信息的累计与重复、散落与零碎、节奏不一与被传播范围所局限在当下互联网几乎普及的今天使我感到十分困惑与窒息。当我选择用最简单的方式来记录这些有意义的、启发我的信息时，我发现现有的最简易最原始的存储信息的方式也是最无用的。在被国内某社交网站上移除了无数个账号后我彻底流浪在互联星河中，像一只脖子上系着铃铛但却浑身沾满泥巴的流浪猫无家可归。我的博客的建立倒不如说是被逼到无路可走，但也的确是在互联网找到靠谱又安全的托管星球的最原始最传统的方法。

我的博客原理

毫无编程经验甚至在决定搭建博客之前都不知道Github是什么的小白如我在这方面有着不同于专业程序员的整套说法。比起照搬Hexo官网的简介与说明或是教科书般死板的定义，我更愿意贴合实际以及用那些不抽象的例子给正在Google搜索栏里输入【Hexo博客搭建教程】的新手们。搭建Hexo到使用的过程(搭建、更换主题、发布文章、学习Markdown语法，维修升级等)就好比从零房屋建造即使“从零”并不严谨，毕竟你已经拥有了联网的计算机。

教程选择建议

首先必须说明用Hexo搭建博客需要一定的编程经验，所以对于纯小白来说并不会是很轻松的体验。在我只靠自己搭建博客的初期曾浏览了网路上许多鱼龙混杂的文字教程，这些文字教程的大致内容虽然相仿但细分到其中的步骤顺序就显得出入很大，更无奈的是很多术业有专攻的程序员作者的表达能力实在堪忧，大部分文字教程只给我带来了干扰与困惑。相比之下我更推荐与我面临相同困境的新手直接跟着Youtube上的视频教程搭建博客，视频教程的优势在于完整地还原了搭建博客的每一个步骤，不需要根据文字凭空想象或小心翼翼地实操生怕出错。凭借视频教程的帮助我在很短的时间内就顺利搭建完成了自己的博客雏形，那时我才意识到自己在文字教程上浪费了太多的时间，同样也再次遗憾自己没有好的导师带领入门以及传授经验避免将时间浪费在踩雷掉坑之上。

Hexo博客搭建详细教程|Windows

安装Node.js与Git

  • Node.js本需不低于10.13 建议使用Node.js12及以上版本 推荐LTS版本
  • Git安装时一路Next即可

    安装Hexo

  • 搜索电脑应用命令提示符
  • 在命令提示符中输入如下代码 即使用npm安装Hexo
    npm install -g hexo-cli

如果你卡在这步屏幕上没有出现+ hexo-cli@x.x.0则代表你没有安装成功，没有安装成功是由于中国大陆防火墙将中国大陆与世界分隔开来所以Node.js时不时失效，尽量确保你的电脑保持可以翻墙的状态。如果电脑已经处于可以使用谷歌看油管的状态却依旧卡在这步，请按照以下教程使用Node.js镜像版。

  • 搜索电脑应用命令提示符
  • 在命令提示符中输入如下代码 安装Node.js镜像版
    npm config set registry https://registry.npm.taobao.org
  • 在命令提示符中输入如下代码 使用npm安装Hexo
    npm install -g hexo-cli
    出现【+ hexo-cli@x.x.0】则代表Hexo初步安装成功！

  • 在命令提示符中输入hexo init nameofyourblog 以我自己的博客名字“Toodles”为例 我应输入：
